Supabase & PostgresML quickstart

Create a managed Postgres backend with TLS enabled:

  1. Launch the provided docker-compose.supabase.yml stack to bootstrap Supabase locally.
  2. Install the PGML extension in the Supabase database using the helper container defined in the compose file.
  3. Set DATABASE_URL, PGML_TLS_MODE=require and a strong ADMIN_API_TOKEN in your environment before starting the LLM server.

The TinyLlama startup model is available out-of-the-box so you can validate the full ingestion workflow without downloading multi-gigabyte checkpoints.

Need a managed backend? Provision a TLS-enabled instance via Supabase and set PGML_TLS_MODE=require before opening this dashboard.
